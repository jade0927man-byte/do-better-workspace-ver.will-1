> 기반: Pyramid Principle (Minto, 1987), SCQA, So What / Now What

## 1. 피라미드 원칙 (Pyramid Principle)

**출처**: Barbara Minto (1987). The Pyramid Principle: Logic in Writing and Thinking. Pearson
**배경**: McKinsey & Company 최초의 여성 컨설턴트 Barbara Minto가 체계화. 컨설팅 업계 표준 보고 구조

**핵심**: 결론부터 말하고, 근거로 뒷받침

### 피라미드 구조

```
                    ┌─────────────┐
                    │   결론/주장  │  ← 핵심 메시지 (1문장)
                    └─────────────┘
                          │
          ┌───────────────┼───────────────┐
          v               v               v
    ┌───────────┐   ┌───────────┐   ┌───────────┐
    │  근거 1   │   │  근거 2   │   │  근거 3   │  ← 핵심 근거 (3개)
    └───────────┘   └───────────┘   └───────────┘
          │               │               │
       세부데이터       세부데이터       세부데이터    ← 상세 분석
```

### 3가지 원칙

```
1. 결론 먼저 (Answer First)
   - 바쁜 경영진은 결론부터 원함
   - "2월 전채널 매출 2.4억, 전월 대비 47% 성장" (O)
   - "여러 데이터를 분석한 결과..." (X)

2. 근거는 MECE하게
   - Mutually Exclusive: 중복 없이
   - Collectively Exhaustive: 누락 없이

3. 3의 법칙 (Rule of Three)
   - 핵심 포인트는 3개로 요약
   - 사람은 3개까지 기억함 (Miller, 1956)
```

> MECE 출처: McKinsey & Company 내부 방법론. Rasiel, E. (1999). The McKinsey Way. McGraw-Hill

## 2. SCQA 프레임워크

**출처**: Barbara Minto (1987). The Pyramid Principle. 스토리라인 구성 기법
**활용**: 보고서 도입부, 프레젠테이션 오프닝

### SCQA 구조

| 요소 | 영문 | 역할 | 뉴믹스 예시 |
|------|------|------|-------------|
| S | Situation | 배경/현황 | "2월 전채널 매출 2.4억원" |
| C | Complication | 문제/변화 | "외국인 비중 83%로 인바운드 의존 심화, 베스트셀러 재고 7일 미만" |
| Q | Question | 핵심 질문 | "3월 벚꽃 시즌에 어떻게 대응할 것인가?" |
| A | Answer | 해결책/결론 | "3가지 액션: 긴급 발주, 국적별 마케팅, 채널 최적화" |

### 작성 순서

```
실제 작성은 역순:
1. Answer 먼저 결정 (결론이 뭔가?)
2. Question 도출 (어떤 질문에 답하는 건가?)
3. Complication 설정 (왜 이 질문이 중요한가?)
4. Situation 배경 (청중이 알고 있는 맥락)
```

## 3. So What / Now What 기법

**출처**: 컨설팅 업계 표준 기법. Zelazny, G. (2001). Say It with Charts. McGraw-Hill

**핵심**: 데이터 → 의미 → 행동으로 연결

### 3단계 사고

```
Data (데이터)
    ↓ So What? (그래서 뭐?)
Insight (인사이트)
    ↓ Now What? (그래서 어쩌라고?)
Action (액션)
```

### 뉴믹스 적용 예시

```
Data: 일본 고객 매출 비중 35%, 객단가 52,000원
    ↓ So What?
Insight: 일본 고객이 최대 매출원이며 고가 상품 선호. 벚꽃 시즌 일본 관광객 증가 시 매출 임팩트 큼
    ↓ Now What?
Action: 일본어 POP/SNS 강화, 일본 고객 선호 상품 재고 우선 확보, 벚꽃 한정 패키지 기획
```

### 안티패턴 (피해야 할 것)

```
X "매출이 47% 올랐습니다" (So What 없음)
X "외국인이 83%입니다" (Now What 없음)
X "데이터를 분석했습니다" (Data도 없음)

O "매출 47% 성장은 외국인 관광객 증가가 주도(So What).
   벚꽃 시즌 추가 30% 증가 예상되므로 재고 선제 확보 필요(Now What)"
```

## 4. 숫자 표현 원칙

**출처**: Zelazny, G. (2001). Say It with Charts. McGraw-Hill
**보충**: Duarte, N. (2010). Resonate: Present Visual Stories. Wiley

### Rule of 3

```
핵심 숫자는 3개 이하:
- 기억하기 쉬움
- 집중도 높음
- 의사결정 가능

좋은 예: "매출 2.4억, 성장률 47%, 외국인 83%"
나쁜 예: "매출 2.4억, 주문 5,711건, 객단가 42,000원, 성장률 47%, 외국인 83%, 일본 35%..."
```

### 숫자 표현법

| 상황 | 표현 | 예시 |
|------|------|------|
| 큰 숫자 | 단위 변환 | 99,000,000원 → 9,900만원 |
| 비교 | 배수/비율 | 47% 증가 → 약 1.5배 |
| 맥락 제공 | 벤치마크 | "업계 평균 10% 대비 47%" |
| 변화 강조 | 차이 | "+13%p" (퍼센트 포인트) |

## 5. 1페이지 보고서 구조

**출처**: Minto (1987) Pyramid + McKinsey 1-page format

### Executive Summary 템플릿

```
┌─────────────────────────────────────────────┐
│  제목: [핵심 메시지를 담은 제목]             │
│  (나쁜 예: "2월 실적 보고")                  │
│  (좋은 예: "2월 전채널 2.4억, 벚꽃 시즌     │
│           재고 확보가 관건")                  │
├─────────────────────────────────────────────┤
│  핵심 숫자 (3개)                            │
│  ┌──────┐  ┌──────┐  ┌──────┐              │
│  │ 2.4억 │  │ +47% │  │ 83%  │              │
│  │전채널  │  │ MoM  │  │외국인 │              │
│  └──────┘  └──────┘  └──────┘              │
├─────────────────────────────────────────────┤
│  So What (인사이트 3개)                     │
│  1. [첫 번째 인사이트]                      │
│  2. [두 번째 인사이트]                      │
│  3. [세 번째 인사이트]                      │
├─────────────────────────────────────────────┤
│  Now What (액션 아이템 3개)                 │
│  1. [액션 + 담당 + 기한]                    │
│  2. [액션 + 담당 + 기한]                    │
│  3. [액션 + 담당 + 기한]                    │
├─────────────────────────────────────────────┤
│  [차트 1개: 가장 임팩트 있는 시각화]        │
└─────────────────────────────────────────────┘
```

### 슬라이드 제목 작성법

```
나쁜 제목: "2월 매출 현황"
좋은 제목: "2월 전채널 매출 2.4억, MoM 47% 성장"

나쁜 제목: "고객 분석"
좋은 제목: "일본 고객이 매출 35% 차지, 벚꽃 시즌 핵심 타깃"

원칙: 제목만 읽어도 메시지 전달
출처: Zelazny (2001) - "Every chart needs an action title"
```

## 6. 보고서 체크리스트

### 작성 전

```
- [ ] 청중(CEO/임원)이 알고 싶은 것은?
- [ ] 의사결정에 필요한 정보가 있는가?
- [ ] 시간 제약은? (5분/15분/30분)
- [ ] Answer(결론)을 먼저 정했는가?
```

### 작성 후

```
- [ ] 첫 문장만 읽어도 핵심 메시지 전달?
- [ ] 핵심 숫자가 3개 이하?
- [ ] 모든 데이터에 So What이 있는가?
- [ ] 모든 인사이트에 Now What이 있는가?
- [ ] MECE하게 구성됐는가?
- [ ] 차트 제목이 메시지를 담고 있는가?
```
