오늘 날짜의 Daily Note를 생성하거나 열어주세요.

**Daily Note는 모든 기록의 허브입니다:**
- 할 일, 메모, 아이디어, 미팅 기록 등 모든 것을 여기에 적습니다
- 할 일이 생각나면 Daily Note의 할 일 섹션에 추가합니다

**수행할 작업:**

1. 오늘 날짜 확인 (YYYY-MM-DD 형식)
2. 경로:
   - 월별 폴더: `./40-personal/41-daily/YYYY-MM/`
   - 파일 경로: `./40-personal/41-daily/YYYY-MM/YYYY-MM-DD.md`
   - 월별 폴더가 없으면 먼저 생성
3. 파일이 없으면:
   - 템플릿 읽기:
     - `./00-system/01-templates/daily-note-template.md`
   - 변수 치환:
     - `{{date}}`: 오늘 날짜 (YYYY-MM-DD)
     - `{{weekday}}`: 오늘 요일 (한국어)
     - `{{yesterday}}`: 어제 날짜 (YYYY-MM-DD)
     - `{{tomorrow}}`: 내일 날짜 (YYYY-MM-DD)
     - `{{week}}`: 이번 주 (YYYY-WXX)
   - 새 파일 생성
4. 파일이 있으면:
   - 현재 내용을 짧게 요약해 표시 (전체 덤프 X — 오늘 우선순위·미완료 핵심만)
   - **이어서 아래 "대화 맥락 자동 추출·반영"을 기본 실행** (사용자가 "업데이트" 명시하지 않아도)

**대화 맥락 자동 추출·반영 (기본 동작):**

`/daily-note` 호출만으로 이번 대화 맥락을 자동 반영합니다.

- 이번 대화에서 나온 내용을 분석하여 자동으로 추출:
  - **완료한 일**: 이번 대화에서 실제로 완료한 작업 → 완료한 일 섹션에 추가
  - **메모/인사이트**: 중요한 결정, 아이디어, 발견 → 메모 섹션에 추가
  - **프로젝트 업데이트**: 프로젝트 관련 진척 → 프로젝트 섹션에 추가
  - **할 일** (제한적 자동 등록): 아래 두 신호 중 하나에 해당할 때만 `- [ ]`로 추가
    1. **명시적 마감일 패턴** — `MM/DD까지`, `내일까지`, `W## 전`처럼 날짜 + 시한 표현이 붙은 발화
    2. **사용자 명시 시그널** — "이거 할일에 넣어줘", "todo로 등록", "추적해줘", "추가해줘" 같은 직접 요청
  - 둘 다 아니면 → 후속 액션이 떠올라도 todo 변환 X. 진척·메모 섹션에만 기록 (정보로만)
- 이미 Daily Note에 있는 내용은 중복 추가하지 않음
- **반영할 내용이 전혀 없으면**: "반영할 신규 맥락 없음. 파일만 표시했습니다." 한 줄로 종료 (불필요한 노이즈 금지)

**영역 진척 자동 분기 (반영 내용이 있을 때 수행):**

하나의 대화에서 여러 영역 진척이 발생하면, 이 커맨드가 **Daily Note + 영역 `progress/`**를 동시에 채웁니다.

**`logs/` vs `progress/`**:
- `logs/` = 1차 원본 기록물. `/daily-note`는 **여기를 건드리지 않음**.
- `progress/` = 진척 스냅샷. `/daily-note`가 여기에 append.

### ⚡ 핵심 원칙: 자동 실행 기본

- **영역 감지 → 즉시 실행** (파일 생성/append)
- **결과 통보로 끝** (승인 대기 X)
- 오분류는 **사후 정정** — 결과 보고 후 수동 삭제 또는 `/progress {올바른영역} {메모}`로 재기록

### 1단계. 영역 감지 → 즉시 실행

1. 대화 맥락에서 영역별 진척·결정·이슈를 감지 (아래 "영역 매핑 테이블" 사용)
2. 각 영역에 대해 곧바로:
   - `{영역폴더}/progress/YYYY-MM-DD.md` 생성 or `## 상세`에 append
   - front-matter `date`, `area`, `tags` (결정/합의→`[결정]`, 이슈/문제→`[이슈]`, 기본 `[진척]`)
   - 본문은 대화에서 추출한 진척 요점 + 결정 근거 (2~5줄)
3. Daily Note 하단 `## 오늘의 진척` 섹션에 링크 append (없으면 신설, 중복이면 스킵)
4. 결과를 사용자에게 통보

**결과 통보 포맷**:
```
영역별 진척 분배 완료:
✓ projects — 10-projects/{프로젝트}/progress/YYYY-MM-DD.md (한 줄 요약)
✓ workspace — 00-system/progress/YYYY-MM-DD.md (한 줄 요약)
```

### 2단계. 영역 매핑 테이블 (기본 골격 — 본인 워크스페이스에 맞게 추가하세요)

이 워크스페이스는 Johnny Decimal 폴더 구조를 씁니다. 아래는 **기본 영역**이고,
본인 사업·프로젝트가 생기면 행을 추가하면 됩니다.

| 영역 | 폴더 | 파일명 | 비고 |
|------|------|-------|------|
| `workspace` | `00-system/progress/` | `YYYY-MM-DD.md` | 도구·설정 튜닝 등 메타 작업 전용 |
| `projects` | `10-projects/{프로젝트}/progress/` | `YYYY-MM-DD.md` | 시한부 프로젝트별 |
| `operations` | `20-operations/{영역}/progress/` | `YYYY-MM-DD.md` | 지속 운영 업무별 |
| `personal` | `40-personal/progress/` | `YYYY-MM-DD.md` | 개인 성장·성찰만 (일정·할일·아이디어는 각자 자리 유지) |

> 📝 **여기를 채우세요**: 본인 사업/프로젝트 영역을 위 표에 추가하세요.
> 예) 카페를 운영한다면 `cafe | 20-operations/{cafe}/progress/ | ...` 행 추가.

### 3단계. 영역 자동 분류 키워드 (본인 영역 키워드를 채우세요)

대화에서 아래 키워드가 감지되면 해당 영역으로 분류합니다.
**처음엔 비어 있습니다 — 본인 업무 키워드를 직접 채워 넣으세요.** 채울수록 분류 정확도가 올라갑니다.

```
# 예시 (본인 사업에 맞게 교체):
# myproject: "프로젝트명", "관련 키워드1", "관련 키워드2"
# operations: "운영 업무 키워드", "거래처명", "정기 작업명"
# personal:   "마음챙김", "성찰", "개인 목표"
```

#### workspace (도구 튜닝)

- `.claude/agents/`, `.claude/commands/`, `.claude/skills/` 관련 작업
- "에이전트", "커맨드", "스킬" 시스템 손보기
- `CLAUDE.md` 수정, "워크스페이스 규칙", 폴더 구조 재편
- "로그 체계", "스키마", "네이밍 규칙"

**예외**: 특정 사업에 종속된 도구(예: "○○ 글쓰기 에이전트 추가")는 해당 사업 영역 우선.

#### 복수 매칭 시

- 명시적 사업 영역 > workspace
- 사업 영역 중 복수 매칭 → 가장 많이 매칭된 영역 (정말 애매하면 양쪽 다 append)

### 4단계. 결과 통보

사용자에게 보고:
- 새로 생성/append된 파일 경로 리스트 (각 1줄 요약)
- 민감정보(실명 등)가 포함된 영역은 **이니셜·요약 처리** 후 강조 표시
- 오분류 복구 방법 안내 (첫 실행 또는 의심 시)

**원칙**:
- **절대 질문 대기로 멈추지 않음** — 답이 안 오면 기본값으로 실행
- 영역 progress 생성이 Daily Note "프로젝트 업데이트" 섹션 작성을 대체하지 않음 — 둘 다 채움

**단순히 즉시 한 줄 기록이 필요하면 `/progress` 커맨드를 쓰세요** — 지정 영역 한 군데에만 기록.

**할 일 자동 동기화 규칙 (제한적):**
- 위 "대화 맥락 자동 추출·반영"의 **두 신호(명시 마감일 / 사용자 명시 시그널) 중 하나에 해당할 때만** 체크박스 항목 추가
- 그 외 후속 액션은 메모·진척 섹션에 정보로만 기록, `- [ ]` 변환 금지
- Daily Note에 `- [ ]` 항목이 추가되면 → `./40-personal/46-todos/todos.md`에도 동기화
- todos.md에 이미 같은 항목이 있으면 중복 추가하지 않음

**사용자가 할 일을 직접 추가 요청하면:**
1. Daily Note의 "할 일" 섹션에 `- [ ]` 형식으로 추가
2. 동시에 `./40-personal/46-todos/todos.md`에도 같은 항목 추가
3. 우선순위가 있으면 🔥(긴급), ⭐(중요) 표시
